# EduChain Kenya: User Manual & Documentation

## 1. Goal
To provide a straightforward guide on compiling, testing, and demonstrating the EduChain Platform.

## 2. Running the Sub-Systems

### 2.1 Backend Server
```bash
cd server
npm install
npm run dev
# Server will start on http://localhost:5000
```

### 2.2 Institution / Admin Web Portal
```bash
cd admin-client
npm install
npm run dev
# Vite server starts on http://localhost:5173
```

### 2.3 Student Mobile/Web App
```bash
cd client
npm install
npm start
# Starts the Expo compiler for Web/Android/iOS
```

## 3. Conducting the Data Exchange Demo

### Scenario: Institution A verifies Certificate generated by Institution B.

**Step 1: Uploading the Certificate (Institution A)**
1. Log into the React Admin portal as "University of Nairobi" (Admin Role).
2. Navigate to the "Add Data" dashboard.
3. Input the student's Registration Index and academic data.
4. *Background Action:* The system creates a hash of this data and submits it to the simulated Blockchain Smart Contract (`CertificateRegistry`).
5. A confirmation states "Data Appended".

**Step 2: Viewing the Certificate (Student)**
1. The student logs into the React Native app.
2. Under "Official Records," the student can see their UoN Degree, stamped as "Blockchain Verified".

**Step 3: Verifying the Certificate (Institution B / Employer)**
1. Another institution accesses the verification endpoint with the student's certificate hash.
2. The smart contract validates the hash signature against the immutable ledger.
3. The dashboard turns Green indicating "Certificate Valid and Authentic," proving successful data exchange without needing a physical phone call.

## 4. Known Challenges Addressed
- **Lack of Standards:** EduChain requires mandatory JSON formatting for all educational data prior to hashing to enforce data uniformity.
- **Scalability:** By keeping large PDFs off-chain (IPFS) and storing only hashes on-chain, transaction costs and network bloat are minimized.
- **Cost:** Using an open-source permissioned setup over public mainnets significantly brings down transaction "gas" costs to near-zero.
